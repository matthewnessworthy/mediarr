# Mediarr Design Context

## Target Audience
Power users who manage large media libraries (movies, TV, anime). They are frustrated with ugly, bloated tools like FileBot and Sonarr. They want something that works correctly and looks beautiful.

## Use Cases
- Scan a folder of downloaded media files, preview proposed renames, and execute
- Monitor folders for new files and auto-rename them
- Review rename history and undo mistakes
- Configure naming templates and subtitle preferences

## Brand Personality
Clean, minimal, precise. Think Linear or Raycast -- lots of whitespace, simple geometry, muted accents. The app should feel like a precision tool, not a media center. Quiet confidence, not flashy.

## Design Constraints
- Dark theme by default, light theme toggle
- Subtle transitions only (expand/collapse, page fades)
- No heavy animations, no spring physics, no bounce/elastic
- shadcn-svelte component library with OKLCH colors
- Tailwind CSS v4 (CSS-first config, no tailwind.config.js)
